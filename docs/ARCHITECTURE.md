# Tquant 量化工作站 - 架构确认与功能覆盖

本架构已明确支持以下两项核心需求，并作为首要设计目标：

## 1. K线图上显示买卖点（BS点）
- **数据源**：回测引擎（`backend/strategy_engine.py`）生成的买卖信号列表（日期、股票代码、方向、价格、数量）。
- **传递方式**：通过 `WebBridge` 以 JSON 格式传给前端。
- **前端渲染**：JavaScript 在 ECharts 散点图上绘制 B/S 标记，位置位于对应日K线的高点上方，颜色区分（红 B / 绿 S），并标注股数与价格。
- **实现文件**：`web/index.html` 中已有相关脚本（`renderKlineWithSignals`），本架构通过桥接对接后端数据后即可工作。

## 2. 策略详情页点击股票代码跳转到该股的买卖点成交图K线
- **交互流程**：
  1. 策略详情页中，股票代码（如 `000001`）以可点击元素呈现。
  2. 点击触发 JS 调用 `webBridge.navigateToKline(code)`。
  3. 后端接收后，切换导航到“买卖点成交图(K线)”页面（左侧导航标志高亮），并传入目标股票代码。
  4. 前端重新渲染对应股票的 K 线及该策略下所有与该股票相关的买卖点信号。
- **数据关联**：每笔信号均包含 `code` 字段，后端可过滤以仅显示特定股票的 BS 点。
- **实现模块**：`app/navigation.py` 管理页面切换，`web/bridge.py` 提供 `navigateToKline` 接口。
